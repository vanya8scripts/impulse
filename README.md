# Импульс

Мессенджер: регистрация с аватаркой, поиск по юзернейму, текст/фото/видео/файлы/голосовые сообщения, аудио- и видеозвонки, темы оформления, статус "в сети", описание профиля.

GitHub Pages отдаёт только статические файлы — своего сервера у него нет. Поэтому для настоящей работы (чтобы два разных человека на двух разных компьютерах реально писали друг другу и звонили) используются два бесплатных внешних сервиса, без банковской карты:

- **Supabase** — база данных, аккаунты, хранилище файлов, доставка сообщений в реальном времени
- **PeerJS Cloud** — бесплатный публичный сигнальный сервер для звонков (ничего настраивать не нужно, уже подключён в коде)

## Шаг 1. Создать проект в Supabase

1. Зайти на **supabase.com** → **Start your project** → войти через GitHub или email (карта не нужна)
2. **New Project** → придумать имя, пароль базы данных (просто сохранить его себе, дальше не понадобится), выбрать любой регион → **Create new project**
3. Подождать 1-2 минуты, пока проект поднимется

## Шаг 2. Выполнить schema.sql

1. В панели проекта слева открыть **SQL Editor**
2. **New query**
3. Открыть файл `schema.sql` из этого архива, скопировать всё содержимое, вставить в редактор
4. Нажать **Run**

Это создаст все таблицы, включит realtime-доставку сообщений, настроит хранилища для аватарок и файлов и пропишет политики доступа (пользователь видит только свои переписки).

## Шаг 3. Взять ключи проекта

1. Слева **Project Settings** (значок шестерёнки) → **Data API**
2. Скопировать **Project URL**
3. Там же, в **API Keys**, скопировать ключ **anon public**

## Шаг 4. Вписать ключи в проект

Открыть файл `js/config.js` и заменить два значения:

```js
window.IMPULSE_CONFIG = {
  supabaseUrl: "https://xxxxxxxxxxxx.supabase.co",
  supabaseAnonKey: "eyJhbGciOiJI...",
  peerCloudKey: "peerjs"
};
```

Сохранить файл.

## Шаг 5. Отключить подтверждение email (важно)

Юзернейм превращается во внутренний технический email вида `vanya@impulse.local`, поэтому подтверждение почты нужно отключить, иначе вход будет заблокирован:

1. **Authentication** → **Providers** → **Email**
2. Выключить переключатель **Confirm email**
3. **Save**

## Шаг 6. Выложить на GitHub Pages

1. Создать новый репозиторий на GitHub (приватный или публичный — на выбор)
2. Загрузить туда все файлы из этого архива, сохранив структуру папок (`css/`, `js/`, `index.html` и так далее)
3. В репозитории: **Settings** → **Pages**
4. **Source** → **Deploy from a branch**, ветка **main**, папка **/ (root)** → **Save**
5. Через минуту GitHub покажет ссылку вида `https://username.github.io/repo-name/` — это и есть рабочий адрес мессенджера

## Готово

Открывай ссылку, регистрируйся, зови друзей регистрироваться по той же ссылке — они находят друг друга по юзернейму через поиск слева, дальше всё работает: переписка, файлы, голосовые, звонки.

## Как это устроено внутри

- Пароли не хранятся в открытом виде — за это отвечает встроенная авторизация Supabase (Auth), проект их даже не видит
- Каждое сообщение защищено политикой (Row Level Security): прочитать переписку может только тот, кто в ней участвует
- Звонки идут напрямую между браузерами (WebRTC) — сообщение о звонке доставляется через базу, а голос/видео не проходят через сервер вообще
- Файлы, фото, видео и голосовые лежат в Supabase Storage и раздаются по прямой ссылке

## Если что-то не работает

- **"Не получилось создать аккаунт" при регистрации** — почти всегда это значит, что в проекте включено подтверждение почты. Зайти в **Authentication → Providers → Email**, выключить **Confirm email**, сохранить и попробовать зарегистрироваться заново. Приложение теперь прямо укажет на эту причину в самой ошибке.
- **Юзернейм, который уже пробовали регистрировать до выключения Confirm email, не проходит** — при первой неудачной попытке аккаунт в Auth мог создаться, а профиль — нет. Зайти в **Authentication → Users**, найти и удалить такого "зависшего" пользователя (email вида `юзернейм@impulse.local`), затем зарегистрироваться заново под тем же юзернеймом
- **"Неверный юзернейм или пароль" при регистрации** — проверить, что в `js/config.js` вписаны правильные `supabaseUrl` и `supabaseAnonKey`, без лишних пробелов
- **Не приходят сообщения в реальном времени** — проверить в Supabase, что в `Database → Replication` таблица `messages` включена (шаг 2 делает это автоматически, но можно перепроверить)
- **Не работает звонок** — браузер должен запросить разрешение на камеру/микрофон, разрешить это; звонки не работают из `file://`, только по `https://` (GitHub Pages отдаёт именно https)
- **Бесплатный тариф Supabase** ограничивает проект после 7 дней полной неактивности (проект "засыпает") — достаточно зайти в панель и нажать "Restore project"
